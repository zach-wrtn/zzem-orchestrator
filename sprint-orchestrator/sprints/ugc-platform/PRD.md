# UGC Platform Sprint PRD

> Original PRD: [docs/prds/PRD-001-ugc-platform.md](../../../docs/prds/PRD-001-ugc-platform.md)

## Scope Summary

ZZEM UGC 플랫폼 구축. 유저가 AI 생성 콘텐츠를 피드에 공개하고, 좋아요/팔로우/크레딧 페이백으로 상호작용하는 소셜 레이어.

## User Stories (7)

### US1: 피드 공개
콘텐츠 생성자가 AI 결과물을 피드에 공개하여 다른 유저에게 노출, 페이백 크레딧 수령.

| AC | 설명 |
|----|------|
| 1.1 | 공개 기본 정책 — 필터 기반 콘텐츠 자동 공개, 기존 콘텐츠 비공개 유지 |
| 1.2 | 페이백 바텀시트 — 재생성 시 1% 크레딧 페이백, 마진 체크 |
| 1.3 | 비공개 전환 — 토글 OFF 시 확인 바텀시트 후 비공개 |
| 1.4 | 커스텀 프롬프트 — 공개 불가, 비공개 탭에만 위치 |
| 1.5 | 풀스크린 비율 — 4:5, 9:16 세로형, cover 크롭 |
| 1.6 | CTA 버튼 분기 — 타인="템플릿 사용하기", 본인="다시 생성하기" |

### US2: 프로필
유저 프로필 관리, 콘텐츠 모아보기, 공유.

| AC | 설명 |
|----|------|
| 2.1 | 프로필 진입점 — MY 탭, 최근 콘텐츠 기준 디폴트 탭 |
| 2.2 | 프로필 구조 — 3탭(게시물/비공개/좋아요), 팔로워/팔로잉/재생성 카운트 |
| 2.3 | 프로필 공유 — OS 공유시트, 딥링크 URL (OG Image 미포함) |
| 2.4 | 프로필 설정 — 프로필 이미지/닉네임 편집, 닉네임 변경 제한 없음 |
| 2.5 | 세로 스와이프 진입 — 탭 내 콘텐츠만 노출 |
| 2.6 | 닉네임 자동 생성 — 최초 진입 시 시스템 자동 생성 |

### US3: 좋아요
콘텐츠에 좋아요, 모아보기.

| AC | 설명 |
|----|------|
| 3.1 | 셀프 좋아요 허용 |
| 3.2 | 좋아요 탭 — 좋아요 시점 최신순, 단일 탭 통합 |
| 3.3 | 추천 시그널 — 좋아요 데이터 추천 전달, 실제 숫자 노출 (0 포함) |

### US4: 크레딧 페이백
공개 콘텐츠 재생성 시 크레딧 페이백.

| AC | 설명 |
|----|------|
| 4.1 | 페이백 조건 — 1% 비율, 마진 체크, 페르소나 제외 |
| 4.2 | 크레딧 히스토리 — "크레딧 페이백" 타이틀 + 썸네일 |
| 4.3 | 세부 정책 — 프로모션 크레딧, 올림 처리, 마진 포함 계산, 최저 0.1% 하한 |

### US5: 알림 & 푸시
좋아요/페이백/팔로우 알림 수신.

| AC | 설명 |
|----|------|
| 5.1 | 알림 카테고리 — 좋아요/크레딧/소식/팔로우 (디폴트 ON) |
| 5.2 | 팔로우 알림 — 즉시 푸시 + 인앱, 활동 알림 카테고리 |
| 5.3 | 알림센터 공통 — 1개월 보관, 미확인 빨간점, 카테고리별 ON/OFF |

### US6: 팔로우
크리에이터 팔로우, 피드 추천 부스트.

| AC | 설명 |
|----|------|
| 6.1 | 팔로우 상태 — 팔로우/팔로잉/맞팔로우 3가지 |
| 6.2 | 팔로우 리스트 — 가나다순, 본인만 조회 가능 |
| 6.3 | 피드 추천 — 팔로우 크리에이터 부스트 가중치 |

### US7: 타 유저 프로필 및 사회적 기능
타 유저 프로필 조회, 차단/신고/의견/페르소나 처리.

| AC | 설명 |
|----|------|
| 7.1 | 타 유저 프로필 — 게시물 탭만, 더보기(URL복사/차단/신고) |
| 7.2 | 차단 — 확인 바텀시트, 편방향 해제, 좋아요 DB 유지/노출 차단 |
| 7.3 | 신고 — 사유 선택 + 자유입력(100자), 추천 패널티, 미통지 |
| 7.4 | 의견 보내기 — 300자 텍스트 + 콘텐츠 ID 첨부, DB 적재 |
| 7.5 | 페르소나 — 알림/페이백 제외, 팔로우 가능, 프로필 동일 UX |

## Total: 7 User Stories, 30 Acceptance Criteria
