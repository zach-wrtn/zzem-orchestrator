# Task 005: 타 유저 프로필 + 생성 후 랜딩

- **Group**: 3
- **AC**: 7.1, 7.5, 2.7

## Target

타 유저 프로필 화면과 생성 후 프로필 랜딩 로직을 구현한다.

## Context

- 타 유저 프로필: 프로필 헤더 + 게시물 탭만 노출 (비공개/좋아요 미노출)
- 진입점: 세로 스와이프 피드의 프로필 영역, 팔로워/팔로잉 리스트
- 더보기 메뉴: 프로필 URL 복사 (차단/신고는 PRD 3)
- 생성 후 랜딩: 밈 생성 완료 후 프로필 화면으로 이동, 생성 타입에 따라 탭 결정
- API: `GET /api/v1/user-profile/:profileId`, `GET /api/v1/content/user/:profileId?visibility=public`

## Objective

타 유저 프로필 화면을 구현하고, 밈 생성 후 프로필 화면 랜딩 로직을 추가한다.

## Specification

### 타 유저 프로필 화면
- 프로필 헤더: 이미지, 닉네임, 팔로워/팔로잉/재생성된 카운트
- 게시물 탭만 노출 (공개 콘텐츠)
- 더보기 메뉴 (⋯): 프로필 URL 복사
- 콘텐츠 탭 → 세로 스와이프 피드 진입 (해당 유저의 게시물만)
- 페르소나 계정: 프로필 진입 시 생성 화면으로 이동하는 동작은 동일

### 생성 후 프로필 랜딩
- 밈 생성 완료 후 → MY 프로필 화면으로 이동
- 필터 기반 생성 → 게시물(공개) 탭으로 랜딩
- 커스텀 프롬프트 기반 생성 → 비공개 탭으로 랜딩
- 생성 중/생성 실패 상태도 해당 탭에서 노출
- 이 규칙은 생성 직후 진입에만 적용, 일반 진입은 AC 2.1 규칙

### Screens / Components
- `OtherUserProfileScreen` — 타 유저 프로필 (ProfileScreen 재사용 가능, 모드 분기)
- 생성 후 랜딩: 네비게이션 파라미터로 `initialTab` 전달

## Acceptance Criteria

1. 세로 스와이프 피드에서 프로필 영역 탭 시 해당 유저의 프로필 화면으로 이동한다
2. 타 유저 프로필에 게시물 탭만 노출된다 (비공개/좋아요 탭 미노출)
3. 타 유저 프로필에 프로필 편집 버튼이 노출되지 않는다
4. 더보기(⋯) 메뉴에 "프로필 URL 복사"가 노출된다
5. 프로필 URL 복사 시 클립보드에 딥링크가 복사된다
6. 타 유저 프로필의 콘텐츠 탭 시 세로 스와이프 피드 진입, 해당 유저 콘텐츠만 노출된다
7. 필터 기반 밈 생성 완료 후 프로필 화면의 게시물 탭으로 랜딩된다
8. 커스텀 프롬프트 기반 생성 완료 후 프로필 화면의 비공개 탭으로 랜딩된다
9. 생성 중/실패 상태가 해당 탭에서 노출된다
10. 페르소나 계정의 프로필 진입 시 생성 화면으로 이동한다

### Implementation Hints

- 프로필 화면 재사용: `isOwnProfile` prop으로 분기
- 네비게이션 파라미터: `{ profileId, initialTab? }`
- SwipeFeed 진입: 기존 SwipeFeedScreen 재사용
