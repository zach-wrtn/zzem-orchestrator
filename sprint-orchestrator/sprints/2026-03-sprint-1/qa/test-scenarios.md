# Sprint 1 QA Test Scenarios

## 대상: PRD Phase 1 (MVP) — 피드 공개/비공개, 프로필, 크레딧 페이백

---

## TS1: 콘텐츠 공개/비공개

### TS1.1: 공개 → 비공개 전환
- Given: 공개 상태인 내 콘텐츠를 세로 스와이프에서 보고 있을 때
- When: 게시 토글을 OFF
- Then: 비공개 확인 바텀시트 노출
- When: 확인 탭
- Then: 콘텐츠가 비공개로 전환, 피드에서 제거, 프로필 비공개 탭으로 이동

### TS1.2: 비공개 → 공개 전환
- Given: 비공개 상태인 내 콘텐츠
- When: 게시 토글을 ON
- Then: 바텀시트 없이 즉시 공개 전환, 피드에 노출

### TS1.3: 커스텀 프롬프트 공개 시도
- Given: 커스텀 프롬프트로 만든 콘텐츠
- When: 게시 버튼 탭
- Then: "커스텀 프롬프트 콘텐츠는 공개할 수 없습니다" 안내 노출

### TS1.4: 타인 콘텐츠 토글 불가
- Given: 타인의 콘텐츠를 보고 있을 때
- Then: 게시 토글 버튼 미노출 또는 비활성

### TS1.5: 기존 콘텐츠 상태
- Given: 기능 업데이트 이전에 생성된 콘텐츠
- Then: 비공개 상태 유지

## TS2: 프로필

### TS2.1: 내 프로필 조회
- Given: 로그인 상태
- When: MY 탭 진입
- Then: 프로필 이미지, 닉네임, 팔로워/팔로잉/재생성 카운트 노출
- Then: 3개 탭 (게시물/비공개/좋아요) 표시

### TS2.2: 타인 프로필 조회
- Given: 피드에서 타인 프로필 탭
- Then: 게시물 탭만 노출, 비공개/좋아요 탭 미노출

### TS2.3: 프로필 콘텐츠 탭 → 세로 스와이프
- Given: 프로필 게시물 탭에서
- When: 콘텐츠 탭
- Then: 세로 스와이프 피드 진입, 해당 탭 콘텐츠만 노출

### TS2.4: 프로필 편집
- Given: 내 프로필에서 편집 버튼 탭
- When: 닉네임 변경 + 저장
- Then: 프로필에 변경된 닉네임 반영
- Edge case: 빈 닉네임 → 에러, 21자 이상 → 에러

### TS2.5: 닉네임 자동 생성
- Given: 신규 유저 최초 가입
- Then: 시스템이 닉네임 자동 생성 (예: "활발한코끼리1234")

---

## TS3: 좋아요 (기존 시스템 확장)

### TS3.1: 셀프 좋아요
- Given: 내 콘텐츠 세로 스와이프
- When: 좋아요 버튼 탭
- Then: 좋아요 카운트 +1, 셀프 좋아요 허용

### TS3.2: 좋아요 탭 통합
- Given: 피드/프로필/검색 등 다양한 경로에서 좋아요
- When: 프로필 좋아요 탭 확인
- Then: 모든 좋아요 콘텐츠가 좋아요 누른 시점 최신순으로 통합 표시

### TS3.3: 좋아요 수 표시
- Given: 콘텐츠
- Then: 그리드 카드 + 세로 스와이프 모두에서 실제 숫자 표시 (0도 표시, 축약 없음)

---

## TS4: 크레딧 페이백

### TS4.1: 정상 페이백
- Given: 유저 A의 공개 콘텐츠 (생성 비용 100 크레딧)
- When: 유저 B가 재생성
- Then: 유저 A에게 1크레딧 적립 (100 * 1% = 1, 프로모션 크레딧)

### TS4.2: 소수점 올림
- Given: 생성 비용 150 크레딧
- When: 재생성 시 페이백 = 150 * 1% = 1.5
- Then: 올림 → 2크레딧 적립

### TS4.3: 페르소나 콘텐츠 페이백 스킵
- Given: 페르소나 계정의 콘텐츠
- When: 유저가 재생성
- Then: 페이백 미적립 (수신 계정 없음)

### TS4.4: 셀프 재생성 스킵
- Given: 유저 A의 콘텐츠
- When: 유저 A가 직접 재생성
- Then: 페이백 미적립

### TS4.5: 비공개 콘텐츠 스킵
- Given: 비공개 상태인 콘텐츠
- When: 재생성 (일어나지 않아야 하지만 방어)
- Then: 페이백 미적립

### TS4.6: 최초 공개 시 페이백 안내
- Given: 유저가 첫 콘텐츠를 생성 + 공개
- Then: 1% 페이백 안내 모달 노출
- When: 확인
- Then: 피드 전환 CTA
- Then: 2번째 공개 시 모달 미노출

## TS5: 피드 CTA 분기

### TS5.1: 타인 콘텐츠 CTA
- Given: 세로 스와이프에서 타인 콘텐츠
- Then: CTA = "템플릿 사용하기"
- When: 탭
- Then: 해당 필터/프리셋으로 새 생성 화면 진입

### TS5.2: 내 콘텐츠 CTA
- Given: 세로 스와이프에서 내 콘텐츠
- Then: CTA = "다시 생성하기"
- When: 탭
- Then: 동일 필터/프리셋으로 재생성
