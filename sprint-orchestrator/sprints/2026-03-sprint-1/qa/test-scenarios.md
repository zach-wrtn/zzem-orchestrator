# QA Test Scenarios — Sprint 2026-03-sprint-1

> PRD-001 UGC Platform 통합 QA 시나리오

## US1: 피드 공개

### TS-1.1: 공개/비공개 토글
- [ ] 필터 콘텐츠 생성 후 자동 공개 상태 확인
- [ ] 공개 콘텐츠 → 비공개 전환 시 확인 바텀시트 노출
- [ ] 비공개 전환 후 피드에서 제거 확인
- [ ] 비공개 → 공개 재전환 가능 확인
- [ ] 기존(업데이트 전) 콘텐츠는 비공개 상태 유지 확인

### TS-1.2: 커스텀 프롬프트 제한
- [ ] 커스텀 프롬프트 콘텐츠 → 공개 시도 시 400 에러
- [ ] 세로 스와이프에서 게시 버튼 → "커스텀 프롬프트 콘텐츠는 공개할 수 없습니다" 안내

### TS-1.3: CTA 버튼 분기
- [ ] 타인 콘텐츠 → CTA = "템플릿 사용하기" → 필터/프리셋 생성 화면 진입
- [ ] 내 콘텐츠 → CTA = "다시 생성하기" → 동일 필터 재생성
- [ ] 소유자 ID vs 현재 유저 ID 비교 로직 정확성

### TS-1.4: AI 라벨
- [ ] 세로 스와이프 상단 '✨ AI가 만드는 컨텐츠예요' 항상 노출

## US2: 프로필

### TS-2.1: 프로필 진입 및 구조
- [ ] 하단 MY 탭 → 프로필 페이지 이동
- [ ] 프로필 이미지, 닉네임, 팔로워/팔로잉/재생성된 카운트 노출
- [ ] 3개 탭: 게시물(공개), 비공개, 좋아요
- [ ] 최근 생성 콘텐츠 기준 디폴트 랜딩 탭 결정

### TS-2.2: 탭별 콘텐츠
- [ ] 게시물 탭: 공개 콘텐츠만 그리드 노출
- [ ] 비공개 탭: 비공개 콘텐츠만 그리드 노출 (본인만)
- [ ] 좋아요 탭: 좋아요한 콘텐츠, 좋아요 시점 최신순
- [ ] 콘텐츠 탭 → 세로 스와이프 진입 (해당 탭 콘텐츠만)

### TS-2.3: 타 유저 프로필
- [ ] 타 유저 프로필: 게시물 탭만 노출
- [ ] 비공개/좋아요 탭 미노출 확인
- [ ] 팔로워/팔로잉 숫자만 노출 (리스트 조회 불가)

### TS-2.4: 프로필 편집
- [ ] 프로필 이미지 변경 (이미지 선택 → 크롭 → 업로드)
- [ ] 닉네임 변경: 제한 없이 자유롭게 가능
- [ ] 저장 후 즉시 반영 확인

### TS-2.5: 프로필 공유
- [ ] 프로필 공유 버튼 → OS 공유 시트 → 딥링크 URL 포함
- [ ] OG Image 미포함 확인 (단순 딥링크만)

### TS-2.6: 닉네임 자동 생성
- [ ] 최초 앱 진입 시 시스템 닉네임 자동 생성 확인
- [ ] 자동 생성된 닉네임 형식 확인

## US3: 좋아요

### TS-3.1: 셀프 좋아요
- [ ] 내 콘텐츠 세로 스와이프에서 좋아요 → 허용, 카운트 반영

### TS-3.2: 좋아요 표시
- [ ] 그리드 카드 + 세로 스와이프: 실제 숫자 그대로 (축약 없음)
- [ ] 0일 때도 숫자 노출

### TS-3.3: 좋아요 탭 통합
- [ ] 피드/프로필/검색 등 어디서 누른 좋아요든 마이페이지 좋아요 탭에 통합

## US4: 크레딧 페이백

### TS-4.1: 페이백 지급
- [ ] 공개 콘텐츠 재생성 → 원작자에게 생성 비용 1% 크레딧 적립
- [ ] 비율: Config 환경변수로 변경 가능 확인
- [ ] 소수점 처리: 올림 (1.2 → 2크레딧)
- [ ] 크레딧 타입: 프로모션 크레딧

### TS-4.2: 마진 체크
- [ ] 페이백 전 마진 체크 수행 확인
- [ ] 마진 부족 시 자동 차단 없음 (모니터링 대응)

### TS-4.3: 페이백 엣지 케이스
- [ ] 공개→비공개 전환: 이미 지급된 페이백 회수 안 함
- [ ] 비공개→공개 재전환: 페이백 재지급 (추가 적립)
- [ ] 콘텐츠 삭제: 기 지급 페이백 유지
- [ ] 페르소나 콘텐츠 재생성: 페이백 적립 대상 없음

### TS-4.4: 크레딧 히스토리
- [ ] "크레딧 페이백" 타이틀 + 콘텐츠 썸네일 노출

### TS-4.5: 최초 진입 UX
- [ ] 최초 콘텐츠 공개 시 1% 페이백 안내 모달
- [ ] 모달 확인 후 피드 전환 CTA
- [ ] 1회성 노출 (재노출 없음)

## US5: 알림 & 푸시

### TS-5.1: 알림 카테고리
- [ ] 좋아요: 개별 알림, 빈도 제한 없음
- [ ] 팔로우: 건별 즉시 푸시 + 인앱, 빈도 제한 없음
- [ ] 크레딧 페이백: 배치 발송
- [ ] 알림 디폴트: 전부 ON

### TS-5.2: 알림센터
- [ ] 보관 기간: 1개월
- [ ] 미확인 알림 → 빨간점(배지) 노출
- [ ] 알림 카테고리별 ON/OFF 설정

### TS-5.3: 페르소나 제외
- [ ] 페르소나 계정: 푸시/알림센터 발송 대상 제외

## US6: 팔로우

### TS-6.1: 팔로우 상태
- [ ] 팔로우/팔로잉/맞팔로우 3가지 상태 정확 표시

### TS-6.2: 팔로우 리스트
- [ ] 내 프로필 → 팔로워/팔로잉 카운트 탭 → 리스트 (가나다순)
- [ ] 각 유저에 팔로우 상태 버튼 노출
- [ ] 타 유저 프로필: 숫자만 노출 (리스트 조회 불가)

### TS-6.3: 팔로우와 추천
- [ ] 팔로우 시 해당 크리에이터 콘텐츠에 피드 추천 가중치 부여 확인 (시그널 전달)

## US7: 타 유저 프로필 및 사회적 기능

### TS-7.1: 타 유저 프로필
- [ ] 프로필 이미지, 닉네임, 팔로워/팔로잉/재생성된 카운트
- [ ] 게시물 탭만 노출
- [ ] 더보기 메뉴: 프로필 URL 복사, 차단하기, 신고하기

### TS-7.2: 차단
- [ ] 차단 확인 바텀시트 → 차단 처리
- [ ] B는 A의 프로필/콘텐츠 접근 불가
- [ ] 팔로우 편방향 해제: A→B만 해제, B→A 유지
- [ ] 차단 사실 미통지
- [ ] 차단된 프로필 진입: "이 계정을 차단했어요" + 차단 해제 버튼
- [ ] 차단 시 좋아요: DB 유지, 노출만 차단, 해제 시 복원

### TS-7.3: 신고
- [ ] 신고 사유 선택 + 자유 입력 (100자, 필수)
- [ ] 신고 사실 미통지
- [ ] 신고한 유저에게 해당 콘텐츠 미노출 (개인화 필터링, ~1시간 지연)

### TS-7.4: 의견 보내기
- [ ] 타 유저 프로필 말풍선 버튼 → 의견 입력 (300자) + 콘텐츠 ID 자동 첨부
- [ ] DB 적재 확인

### TS-7.5: 페르소나 계정
- [ ] 페르소나 팔로우: 가능 (UX 동일)
- [ ] 페르소나에 팔로우 알림 미발송
- [ ] 페르소나 콘텐츠 좋아요: 가능
- [ ] 페르소나 프로필 진입 → 생성 화면 이동 동작 동일
