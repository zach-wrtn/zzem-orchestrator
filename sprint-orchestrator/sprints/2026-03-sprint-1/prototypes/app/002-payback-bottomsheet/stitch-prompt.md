Design a mobile app screen for a Korean AI content creation app called "ZZEM" (짬).

Platform: iOS and Android (React Native)
Language: Korean (한국어) for all UI labels and text
Style: Modern, clean, minimal mobile app design
Theme: Dark theme with vibrant accent colors

---

## Screen: PaybackInfoBottomSheet

### Purpose
First-time publish info modal showing 1% payback explanation. One-time display. Guides user to explore feed after dismissal.

### Layout & Components

- **PaybackInfoBottomSheet**: Modal bottom sheet with an illustration or icon at the top center, explanation text in the middle, a confirm button, and a feed exploration CTA button below it.
  - Illustration/icon: visual representing credits or payback (e.g., coin or sparkle icon)
  - Title: "크레딧 페이백 안내"
  - Body text: "콘텐츠가 재생성될 때마다 생성 비용의 1%가 페이백됩니다"
  - Primary button: "확인" — closes the bottom sheet
  - Secondary CTA button: "피드 둘러보기" — navigates to feed
- **CreditPaybackHistoryItem**: Credit history list item component (shown in the credit history screen, not in this bottom sheet). Displays "크레딧 페이백" as the title with a content thumbnail beside it.

### User Flow

1. User publishes content for the first time — this bottom sheet appears automatically
2. User reads the payback explanation (1% of generation cost returned when content is regenerated by others)
3. Tapping "확인" closes the bottom sheet and returns to the previous context
4. Tapping "피드 둘러보기" closes the bottom sheet and navigates to the feed screen
5. This bottom sheet is never shown again after the first display (one-time)

### Visual Rules

- One-time display only, tracked via local storage flag
- Payback rate (currently 1%) should be displayed dynamically so it can be updated without code changes
- Credit type for payback is promotion credit (may need visual distinction from paid credit if applicable)
- Bottom sheet should have a dimmed backdrop and support drag-to-dismiss
