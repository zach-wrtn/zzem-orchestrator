## ⚠️ Fix Ready for QA — free-filter-qa-fix / group-1

**Root Cause**
`FilterRepository.findOneActive` 의 `activatedAt: { $lte: new Date() }` 단독 쿼리가 무료 필터처럼 `activatedAt` 미설정 도큐먼트를 단건 조회 시 매치 못 시켜 null → 404. 같은 모듈의 `findActiveByIds` 는 이미 `$or: [{ $lte }, { $exists: false }]` 라 목록은 정상 → 단건/목록 비대칭 버그. FilterPreviewScreen 이 `useSuspenseQuery` 로 호출하므로 404 가 throw 되어 ErrorBoundary 의 에러 화면이 노출됨. "유료 필터에선 정상" 단서는 유료 필터에 `activatedAt` 이 채워져 있어 단독 `$lte` 가드 통과.

**Fix Summary**
`findOneActive` 쿼리 조건을 `findActiveByIds` 와 동일한 `$or: [{$lte}, {$exists: false}]` 로 변경. 비대칭 제거. 무료 필터처럼 `activatedAt` 미설정 케이스도 정상 활성으로 인식되어 프리뷰 진입 시 정상 화면 노출. 한 fix 로 IS-1365/IS-1367 동시 해소.

**Verification Steps**
1. 무료 탭 → 무료 필터 피드 → 프리뷰 버튼 탭 → 프리뷰 화면 정상 노출 (모든 무료 슬롯)
2. 무료 1회 사용 후 다른 무료 필터의 프리뷰 진입 → 정상 노출
3. (회귀) 유료 필터의 프리뷰 진입 → 정상 (변경 없음)
4. (회귀) `isActive: false` / `deletedAt` 보유 / `activatedAt` 미래 시각 필터 → 404 차단 유지
