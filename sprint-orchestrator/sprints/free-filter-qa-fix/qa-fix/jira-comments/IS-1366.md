## Fix Ready for QA — free-filter-qa-fix / group-4

**Root Cause**
홈 추천 탭의 sticky FilterChips overlay 와 HomeHeader 의 웹툰 SimpleTooltip 이 둘 다 `zIndex: 10` 사용 → 같은 stack context 에서 DOM/render 순서에 따라 sticky overlay 의 배경이 tooltip 텍스트 아래로 들어가 영역이 시각적으로 겹침. Tooltip 자체의 zIndex (1000) 는 충분히 높지만 sticky overlay 의 흰 배경 + content 가 같은 layer 에 위치.

**Fix Summary**
home-body 의 `stickyOverlay` 스타일 zIndex 10 → 11 로 1 단계 승급. sticky FilterChips 가 항상 HomeHeader 위에 그려져 겹침 방지. 다른 화면의 ToolTip / FilterChip 사용처는 영향 없음 (`stickyOverlay` 단일 사용처).

**Verification Steps**
1. 홈 추천 탭 진입 (웹툰 툴팁이 노출되는 상태) → 툴팁 정상 표시
2. 추천 탭 하단으로 스크롤 → sticky FilterChips 영역 진입 시 툴팁과 겹침 **없음**
3. 위로 스크롤 → sticky 해제 → 헤더 정상
4. 툴팁 dismiss 후 동일 시나리오 → 정상
