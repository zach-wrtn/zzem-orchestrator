## ⚠️ Fix Ready for QA — free-filter-qa-fix / group-2

**Root Cause**
두 개의 결합:
1. `useCreateWebtoonEpisodeUseCase` 의 mutation 이 `onSuccess` 만 `invalidateMySeries` / `invalidateActiveGenerations` 호출. 실패 경로 (`onError`/`onSettled`) 누락 → backend 가 1화 실패 시 series 를 softDelete 해도 FE 캐시 stale → "내 웹툰" 목록에 orphan series 남음.
2. `WebtoonViewerScreen.chipIndices` 가 `episodeIdMapRef` 의 stale entry 를 정리하는 로직이 `episode.status === "failed"` trigger 에만 의존. 사용자가 다른 화로 수동 전환하면 trigger 안 됨 → backend 가 회차를 rollback 해도 미생성 회차 chip 노출.
크레딧 환불은 BE 트랜잭션이 정상 처리 (FE 변경 없음).

**Fix Summary**
두 곳을 보강:
- `useCreateWebtoonEpisodeUseCase` 의 `onSuccess` → `onSettled` 로 변경. 성공/실패 모두 캐시 invalidate → orphan series 제거.
- `chipIndices` 계산 시 `episodeListData` (server truth) 와 `episodeIdMapRef` 를 cross-check. server 에 없는 stale entry 제외. `episodeListData` 빈 fallback (자동이어가기 직후 fetch 전) 은 보존.

**Verification Steps**
1. 웹툰 1화 정상 생성 → "내 웹툰" 목록 등록 (회귀 점검)
2. 웹툰 1화 실패 (네트워크 차단 또는 비윤리 이미지) → "내 웹툰" 목록에 entry **없음**
3. 1화 정상 후 → 2화 자동이어가기 실패 → 회차 chip 에 2화 표시 **안 됨** (1화만)
4. 2.의 실패 후 같은 회차 재시도 → 정상 진입
5. 크레딧 영수증 (CreditHistory) → 환불 정상 (BE 변경 없음 확인)
