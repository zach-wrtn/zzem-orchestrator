# 1. Overview

## 한 줄 요약

- 유저가 ZZEM 에서 제공한 필터로 생성한 콘텐츠를 피드에 공개하고,
- 다른 유저와 좋아요·팔로우·크레딧 페이백으로 상호작용하는 UGC 플랫폼을 구축한다.

## Why (비즈니스 목표)

1. 콘텐츠 풀 확대 — 유저가 만든 콘텐츠를 공개하여 피드 다양성과 볼륨을 높인다
2. 추천 시그널 확보 — 좋아요/팔로우 인터랙션으로 추천 품질을 향상시킨다
3. 재생성 전환 유도 — 타 유저 콘텐츠를 보고 "나도 만들어볼까" 전환을 극대화한다
4. 크레딧 순환 — 페이백으로 크레딧 소비→공개→재생성→재소비 루프를 만든다

## 대상 사용자

- ZZEM 앱 사용자 (AI 이미지/영상 생성 유저)
- 콘텐츠 소비자 — 피드를 탐색하며 다른 유저의 결과물을 감상하고 재생성하는 유저
- 콘텐츠 생산자 — 자신의 결과물을 공개하고 페이백 크레딧을 받는 유저

## KPI 기여 논리

- 피드 공개율 ↑ → 콘텐츠 풀 확대 → 피드 체류시간/재방문율 ↑ → 재생성 전환율 ↑ → 크레딧 소비 ↑
- ARPPU ↑ → 페이백 크레딧 공개 인센티브 → 다시 재생성에 사용 → 크레딧 순환 루프 형성

## 선행 조건

- 피드 & 추천 PRD (배포 완료) — 그리드 피드, 세로 스와이프, 좋아요 UI, 추천 로직 영역 정의
- 기존 크레딧 시스템 — 크레딧 잔액 조회, 차감, 적립 API
- 기존 인증 시스템 — 유저 계정, 프로필 기본 정보

# 2. User Stories & Acceptance Criteria

7개 기능 영역을 User Story로 구성. 각 AC는 Given/When/Then 형식으로 기술하여 AI 코드 생성 시 테스트 코드로 직접 변환 가능하도록 한다.

## US1: 피드 공개

콘텐츠 생성자로서, 내 AI 생성 결과물을 피드에 공개하여 다른 유저들이 볼 수 있게 하고, 페이백 크레딧을 받고 싶다.

### AC 1.1: 공개 기본 정책

- Given 유저가 필터(프리셋)를 사용하여 콘텐츠를 생성 완료했을 때
- When 결과물이 생성되면
- Then 콘텐츠는 자동으로 공개(게시) 상태가 되며, 피드에 노출된다
- Then 콘텐츠에 워터마크/로고 없이 원본 그대로 노출된다
- Then 기능 업데이트 이전에 생성된 콘텐츠는 비공개 상태를 유지한다

### AC 1.2: 페이백 바템시트

- Given 콘텐츠가 공개 상태일 때
- When 해당 콘텐츠가 다른 유저에 의해 재생성되면
- Then 원콘텐츠 소유자에게 생성 비용의 1% 크레딧이 페이백된다 (Config 환경변수로 비율 변경 가능)
- Then 페이백 전 마진 체크를 수행하여 적자 방지

### AC 1.3: 비공개 전환

- Given 콘텐츠가 공개 상태일 때
- When 유저가 게시 토글을 OFF하면
- Then 비공개 전환 확인 바템시트가 노출된다
- Then 확인 시 콘텐츠가 비공개로 전환되고 피드에서 제거된다

### AC 1.4: 커스텀 프롬프트 결과물

- Given 커스텀 프롬프트 기반으로 생성된 콘텐츠일 때
- Then 피드 공개(게시) 불가 — 프로필 비공개 탭에만 위치
- Then 세로 스와이프에서 게시 버튼 클릭 시 "커스텀 프롬프트 콘텐츠는 공개할 수 없습니다" 안내 노출
- Then Phase 2에서 커스텀 프롬프트 기반 UGC 피드 공개 지원 예정

### AC 1.5: 풀스크린 비율 정책

- Given 피드 콘텐츠가 풀스크린으로 표시될 때
- Then 지원 비율: 4:5, 9:16 등 세로형 비율을 지원한다
- Then 풀스크린 표시 방식: cover (크롭) — 콘텐츠가 화면을 꽉 채우며, 비율이 맞지 않으면 크롭
- Then 16:9 디바이스에서 세로형(9:16) 콘텐츠 렌더링 시 좌우를 크롭하여 화면에 맞춤
- Then 세로형 피드에서 좌우 스와이프 시 이미지 간 이격 허용 범위 명시 필요 (디자인 확정 필요)

### AC 1.6: 피드 CTA 버튼 분기

- Given 세로 스와이프 피드에서 콘텐츠 하단 CTA 버튼이 표시될 때
- When 타인의 콘텐츠를 보고 있으면
- Then CTA 버튼 = "템플릿 사용하기" — 해당 필터/프리셋으로 새 콘텐츠 생성 화면 진입
- When 내 콘텐츠를 보고 있으면
- Then CTA 버튼 = "다시 생성하기" — 동일 필터/프리셋으로 재생성
- Then 시스템은 콘텐츠 소유자 ID와 현재 유저 ID를 비교하여 CTA 유형을 분기한다

## US2: 프로필

유저로서, 내 프로필을 관리하고 내 콘텐츠를 모아볼 수 있으며, 다른 유저와 공유할 수 있다.

### AC 2.1: 프로필 진입점

- Given 유저가 로그인된 상태일 때
- When 하단 MY 탭을 탭하면
- Then 프로필 페이지로 이동한다
- Then 최근에 만든 콘텐츠 기준으로 디폴트 랜딩 탭이 결정된다

### AC 2.2: 프로필 구조 및 탭

- Given 프로필 페이지에 진입하면
- Then 프로필 이미지, 닉네임, 팔로워/팔로잉/재생성된 카운트가 노출된다
- Then 3개 탭 구조로 노출된다:
  - 탭 1 — 게시물 (공개): 공개 상태인 내 콘텐츠만 그리드로 노출
  - 탭 2 — 비공개: 비공개 상태인 내 콘텐츠 그리드. 본인만 보임
  - 탭 3 — 좋아요: 내가 좋아요한 콘텐츠 그리드
- Then "재생성된" = 내 공개 콘텐츠가 타 유저에 의해 재생성된 횟수

### AC 2.3: 프로필 공유

- Given 프로필 페이지에서
- When "프로필 공유" 버튼을 탭하면
- Then OS 기본 공유 시트가 호출되며 프로필 URL(딜링크)이 포함된다
- Then 앱 설치 유저는 앱 내 프로필로, 미설치 유저는 브릿지 웹 경유
- Then 프로필 URL 공유 시 OG Image는 포함되지 않으며, 단순 딥링크로만 동작한다 (콘텐츠 공유 시에만 OG Image 포함)

### AC 2.4: 프로필 설정

- Given 프로필 페이지에서
- When "프로필 편집" 버튼을 탭하면
- Then 프로필 이미지와 닉네임을 수정할 수 있다
- Then 닉네임 변경은 제한 없이 자유롭게 가능하다

### AC 2.5: 프로필에서 세로 스와이프 진입

- Given 프로필의 게시물/비공개 탭에서
- When 콘텐츠를 탭하면
- Then 세로 스와이프 피드로 진입하며, 해당 탭의 콘텐츠만 노출된다

### AC 2.6: 닉네임 자동 생성 규칙

- Given 유저가 최초로 앱에 진입하여 프로필이 생성될 때
- Then 시스템이 자동으로 닉네임을 생성한다
- Then 자동 생성 포맷/규칙은 별도 정의 필요 (예: 형용사+랜덤숫자, 영어닉네임 등)
- Then 유저는 나중에 프로필 편집에서 닉네임을 변경할 수 있다

## US3: 좋아요

콘텐츠 소비자로서, 마음에 드는 콘텐츠에 좋아요를 누르고 나중에 모아볼 수 있다.

### AC 3.1: 셀프 좋아요

- Given 내 콘텐츠의 세로 스와이프 피드에서
- When 좋아요 버튼을 탭하면
- Then 셀프 좋아요가 허용되며 좋아요 카운트에 반영된다

### AC 3.2: 좋아요 탭

- Given 마이페이지 > 좋아요 탭에서
- Then 좋아요한 콘텐츠를 좋아요 누른 시점 최신순으로 모아볼 수 있다
- Then 피드, 프로필, 검색 등 어떤 경로에서 누른 좋아요든 단일 탭에 통합 표시된다 (v3 추가)

### AC 3.3: 좋아요 추천 시그널

- Then 좋아요 데이터는 추천 시스템에 시그널로 전달된다
- Then 지나치게 과거 콘텐츠에 좋아요한 것은 추천 로직을 훼손시킬 가능성이 있으므로 고려 필요
- Then 좋아요 수 표시: 그리드 카드 + 세로 스와이프 상세 모두에서 실제 숫자 그대로 노출 (축약 없음, 0일 때도 노출)

## US4: 크레딧 페이백

콘텐츠 생성자로서, 내 공개 콘텐츠가 재생성될 때 크레딧을 페이백 받아 더 많은 콘텐츠를 만들고 싶다.

### AC 4.1: 페이백 조건

- Given 내 공개 콘텐츠가 타 유저에 의해 재생성될 때
- Then 재생성 비용의 1% 크레딧이 내 계정에 적립된다 (비율은 Config 환경변수로 변경 가능)
- Then 페이백 전 마진 체크 로직이 수행되어 적자를 방지한다
- Then 페르소나 계정 콘텐츠가 재생성될 경우 페이백 적립 대상 없음 (수신할 계정 없음)

### AC 4.2: 크레딧 히스토리

- Given 페이백이 발생하면
- Then 크레딧 히스토리에 "크레딧 페이백" 타이틀 + 콘텐츠 썸네일로 노출된다

### AC 4.3: 크레딧 페이백 세부 정책 (v3 추가)

- Given 페이백 크레딧이 적립될 때
- Then 크레딧 타입 = 프로모션 크레딧 (유료 크레딧과 구분)
- Then 소수점 처리 = 올림 후 점수 단위 (예: 1.2크레딧 → 2크레딧 적립)
- Then 마진에서 차감하지 않고 마진에 포함하여 계산 (마진 = 수익 - 원가 - 페이백)
- Then 마진 압박 시 최저 0.1% 하한까지 축소 가능 (Config로 조정)

### AC 4.4: 최초 진입 UX (페이백 안내) (v3 추가)

- Given 유저가 콘텐츠를 최초로 생성하고 공개했을 때
- Then 1% 페이백 안내 모달이 노출된다: "콘텐츠가 재생성될 때마다 생성 비용의 1%가 페이백됩니다"
- Then 모달 확인 후 피드 전환 CTA 노출 (피드 탐색 유도)
- Then 1회성 노출 (최초 공개 시에만)

## US5: 알림 & 푸시

유저로서, 내 콘텐츠에 대한 좋아요, 페이백, 팔로우 등의 알림을 받아 소식을 확인하고 싶다.

### AC 5.1: 알림 카테고리

- Then 좋아요 알림, 크레딧 알림, 소식 알림, 팔로우 알림 카테고리가 제공된다 (디폴트 전부 ON)
- Then 좋아요 알림: 빈도 제한 없음 (개별 알림)
- Then 팔로우 알림: 건별 발송, 맞팔로우 시 별도 알림 없음. 빈도 제한 없음
- Then 크레딧 페이백 알림: 배치 발송
- Then 페르소나 계정은 푸시/알림센터 발송 대상에서 제외

### AC 5.2: 팔로우 알림

- Given 다른 유저가 나를 팔로우했을 때
- Then 즉시 푸시 알림 + 인앱 알림센터에 노출
- Then '활동 알림' 카테고리에 포함
- Then 알림 내용: 'OOO님이 회원님을 팔로우했습니다'

### AC 5.3: 알림센터 공통

- Then 알림센터 보관: 1개월
- Then 미확인 알림 존재 시 빨간점 노출
- Then MY 탭 설정(톱니바퀴) 아이콘에서 알림 카테고리별 ON/OFF 설정 가능

## US6: 팔로우

유저로서, 마음에 드는 크리에이터를 팔로우하여 그들의 새 콘텐츠를 피드에서 더 쉽게 발견하고 싶다.

### AC 6.1: 팔로우 상태

- Then 팔로우 상태는 팔로우/팔로잉/맞팔로우 3가지로 표시된다

### AC 6.2: 팔로우 리스트

- Given 내 프로필에서
- When 팔로워/팔로잉 카운트를 탭하면
- Then 각각의 리스트가 가나다순으로 노출되며, 각 유저에 팔로우 상태 버튼 노출
- Then 타 유저 프로필에서는 팔로워/팔로잉 숫자만 노출 (리스트 조회 불가, 본인만 가능)

### AC 6.3: 팔로우와 피드 추천

- Given 유저가 특정 크리에이터를 팔로우하면
- Then 해당 크리에이터의 콘텐츠에 피드 추천 가중치(부스트)가 부여된다
- Then 상세 로직은 PA(추천) 팀에서 정의 (별도 전달 필요)

## US7: 타 유저 프로필 및 사회적 기능

유저로서, 다른 유저의 프로필을 보고 팔로우/차단/신고할 수 있다.

### AC 7.1: 타 유저 프로필

- Given 타 유저 프로필에 진입하면
- Then 프로필 이미지, 닉네임, 팔로워/팔로잉/재생성된 카운트 노출
- Then 게시물 탭만 노출 (비공개/좋아요 탭 미노출)
- Then 더보기 메뉴: 프로필 URL 복사, 차단하기, 신고하기

### AC 7.2: 차단

- Given 유저 A가 유저 B를 차단하면
- Then 차단 확인 바템시트 노출 후 차단 처리
- Then B는 A의 프로필/콘텐츠 접근 불가
- Then 팔로우 편방향 해제: A→B 팔로우만 해제, B→A 팔로우는 유지
- Then 차단 사실은 상대방에게 미통지
- Then 차단 해제는 언제든 가능
- Then 차단된 프로필 진입 시: "이 계정을 차단했어요" + 차단 해제 버튼
- Then 차단 시 좋아요 데이터: DB에 유지, 노출만 차단. 차단 해제 시 다시 보임

### AC 7.3: 신고

- Then 신고 사유 선택 + 자유 입력 (100자 이내, 필수)
- Then 신고 사실은 상대방에게 미통지
- Then 노출 대비 신고 비율이 높은 콘텐츠에 추천 패널티 적용 (노출 가중치 하향) (v3 추가 - 이번 릴리즈 포함)
- Then 신고한 유저에게는 해당 콘텐츠 미노출 (개인화 필터링) (v3 추가)
- Then 개인화 필터링 반영 지연: ~1시간 (실시간 아님) (v3 추가)
- Then 신고 비율 임계값 및 패널티 강도는 PA(추천) 팀과 협의 필요 (v3 추가)

### AC 7.4: 의견 보내기

- Given 타 유저 프로필에서
- When 팔로우 버튼 옆 독립 말풍선 아이콘 버튼을 탭하면 (Figma 기준)
- Then 의견 보내기 화면으로 이동
- Then 자유 텍스트 입력 (300자 이내) + 해당 콘텐츠 ID 자동 첨부
- Then 수집된 의견은 DB에 적재만 하고, 운영 어드민 조회는 후순위

### AC 7.5: 페르소나 계정 처리

- Then 페르소나 계정 = 콘텐츠 팩토리 운영용 데이터 전용 계정 (실제 디바이스/유저 없음)
- Then 계정 레벨 페르소나 여부 식별 플래그/필드 필요 (BE 구현)
- Then 알림 수신 불가, 페이백 적립 대상 없음
- Then 유저가 페르소나를 팔로우 가능 (UX 상 구분 없음), 단 페르소나 측에 팔로우 알림 발송 안 함
- Then 페르소나 콘텐츠 좋아요: 가능
- Then 페르소나 프로필 진입 시 생성 화면으로 이동하는 동작은 동일

# 3. 비즈니스 룰 & 엣지 케이스

v1에 흩어져 있던 비즈니스 룰을 한 곳에 모음. AI 코드 생성 시 이 섹션을 참조하여 구현.

## 콘텐츠 공개 규칙

1. 커스텀 프롬프트 결과물 = 피드 공개 불가 (Phase 2 예정)
2. 기능 업데이트 전 콘텐츠 = 비공개 상태 유지
3. 업데이트 후 콘텐츠 = 공개가 디폴트
4. 콘텐츠 공개 시 워터마크/로고 없이 그대로 노출
5. 세로 스와이프 상단에 '✨ AI가 만드는 컨텐츠예요' 라벨 항상 노출. 투명성 정책으로 제거 불가
6. 콘텐츠 정렬 순서: 내 프로필 전체 탭(3개 탭 모두) 최신순 (생성일 내림차순). 좋아요 탭은 좋아요 누른 시점 최신순

## 페이백 규칙

1. 페이백 비율 = 생성 비용의 1% (Config/환경변수로 변경 가능)
2. 페이백 전 마진 체크 필수 — 적자 방지
3. 공개→비공개 전환 시 페이백: 회수 안 함. 한 번 지급된 페이백 크레딧은 비공개 전환해도 유지됨
4. 비공개→공개 재전환 시 페이백: 재지급 (추가 적립). 해당 콘텐츠에 대해 재공개할 때마다 페이백 새로 적립. ⚠️ 토글 어뷰징 가능성 있으므로 모니터링 필요
5. 마진 부족 시: 시스템 자동 차단 없음. 대시보드 모니터링 후 Config 값 수동 조정으로 대응
6. 공개 콘텐츠 삭제: 삭제 가능. 삭제해도 이미 지급된 페이백 크레딧은 유지됨
7. 어드민 크레딧 회수: 어드민 패널에서 수동으로 크레딧 회수 가능. 크레딧 히스토리에 '크레딧 회수' 항목으로 노출
8. 페르소나 콘텐츠 재생성 시 페이백 적립 대상 없음 (수신 계정 없음)
9. 크레딧 타입 = 프로모션 크레딧 (유료 크레딧과 구분) (v3 추가)
10. 소수점 처리 = 올림 후 점수 단위 (v3 추가)
11. 마진에서 차감하지 않고 마진에 포함하여 계산 (v3 추가)
12. 마진 압박 시 최저 0.1% 하한까지 축소 가능 (Config로 조정) (v3 추가)

## 좋아요 규칙

1. 셀프 좋아요 허용
2. 과거 콘텐츠 좋아요가 추천 로직에 미치는 영향 고려 필요
3. 좋아요 수 표시: 그리드 카드 + 세로 스와이프 상세 모두에서 실제 숫자 그대로 노출 (축약 없음, 0일 때도 노출)
4. 페르소나 콘텐츠 좋아요: 가능

## 팔로우 규칙

1. 차단 시 팔로우 편방향 해제: A가 B를 차단하면 A→B만 해제, B→A 유지
2. 팔로우한 사람의 콘텐츠 = 피드 추천에 부스트 반영
3. 페르소나 계정 팔로우: 가능. 프로필 진입 시 생성 화면으로 이동하는 동작 동일

## 알림 규칙

1. 알림센터 보관: 1개월
2. 페르소나 계정은 모든 알림 수신 대상에서 제외
3. 팔로우 알림: 활동 알림 카테고리. 즉시 푸시 + 인앱. 빈도 제한 없음
4. 좋아요 알림: 빈도 제한 없음 (개별 알림)
5. 크레딧 페이백 알림: 배치 발송

## 차단 규칙

1. 차단 시 좋아요 데이터: DB에 유지, 노출만 차단. 차단한 유저의 콘텐츠가 피드/프로필에서 안 보임. 차단 해제 시 다시 보임

## 프로필 규칙

1. 닉네임 변경: 제한 없이 자유롭게 가능 (v3 변경: 7일 제한 삭제)
2. 최초 진입 시 시스템이 닉네임 자동 생성 (v3 추가)

## 콘텐츠 공유 규칙

1. 공유 방식: 딥링크 공유. 카카오톡 등에서 열면 OG Image로 프리뷰 노출
2. OG Image: 인하우스 콘텐츠 = 필터 프리뷰 크롭, UGC = 유저 결과물 프리뷰 크롭
3. OG title/description: 원본 인하우스 필터의 정보를 상속받음
4. 딥링크 랜딩: 앱 설치 시 해당 콘텐츠로 직행, 미설치 시 앱 스토어로 리다이렉트
5. 프로필 공유 시 OG Image 미포함 — 단순 딥링크로만 동작 (v3 추가)

## 기존 밈 마이그레이션 (v3 추가)

1. 기존 생성 밈 → 비공개 탭으로 일괄 마이그레이션
2. 마이그레이션 시점: 기능 배포 시 일괄 처리 (배치)
3. 마이그레이션 대상: 기존 밈 생성 API로 생성된 모든 콘텐츠
4. 마이그레이션 후 콘텐츠는 프로필 비공개 탭에서 조회 가능

## 그리드 뮤트 콘텐츠 비중 (v3 추가)

1. 그리드 피드에서 뮤트(무음) 콘텐츠의 노출 비중은 매출 극대화 기준으로 설정
2. 세부 비중 조정은 추후 데이터 기반으로 결정 (Config로 관리)

# 4. 경계 (3-Tier Boundary)

AI 코드 생성 시 판단 기준. 불확실하면 ASK FIRST 영역으로 분류하여 PM에게 확인한다.

## ALWAYS DO (항상 지킬 것)

1. 콘텐츠 공개/비공개 전환 시 확인 바템시트 노출
2. 페이백 전 마진 체크 로직 수행
3. 커스텀 프롬프트 콘텐츠는 공개 불가 처리
4. 차단/신고 사실은 상대방에게 미통지
5. 페르소나 계정은 알림/페이백 대상 제외
6. 기존 크레딧 시스템 API를 통한 적립/차감
7. 기존 인증 시스템을 통한 유저 식별
8. 공개→비공개 전환 시 페이백 크레딧 회수 안 함
9. 비공개→공개 재전환 시 페이백 재지급 (추가 적립)
10. AI 콘텐츠 라벨 항상 노출

## ASK FIRST (PM에게 확인 후 진행)

1. 페이백 비율 변경 (Config 값 확인)
2. 추천 로직 가중치 세부 튜닝 (PA팀 영역)
3. 알림 문구/포맷 변경
4. 신규 푸시 카테고리 추가
5. OG Image/딥링크 랜딩 정책 변경

## NEVER DO (절대 하지 말 것)

1. DM/메시지 기능 구현
2. 검색/해시태그 기능 구현
3. 수익화/광고 로직 구현
4. 콘텐츠 모더레이션 시스템 구현
5. 팔로워 피드/타임라인 구현 (추천 피드에 팔로우 부스트만 반영)
6. 추천 알고리즘 코어 로직 직접 변경 (v3 수정: 신고 패널티/팔로우 부스트는 시그널 전달로 구현, 코어 로직만 금지)
7. 웹 버전/딥링크 랜딩 페이지 구현
8. 댓글 고도화 기능 구현

# 5. 기존 맥락

AI 코드 생성 시 기존 시스템과의 연동 지점을 파악하기 위한 섹션.

## 선행 PRD

- 피드 & 추천 PRD (배포 완료) — 그리드 피드, 세로 스와이프, 좋아요 UI 위치/인터랙션, 추천 로직 적용 영역 정의. 본 PRD의 좋아요/피드 기능은 이 PRD 위에 레이어링된다.

## 기술 스택

- Git 레포 참조 — wrtn-backend/meme-api (BE), app-core-packages (FE). PRD에 기술 스택은 생략하며 코드베이스에서 직접 확인한다.

## 기존 시스템 연동

- 크레딧 시스템 — 잔액 조회, 차감, 적립 API. 페이백은 이 시스템을 통해 적립
- 인증 시스템 — 유저 계정, 프로필 기본 정보. 프로필 확장은 이 시스템 위에 구축
- 추천 시스템 — PA팀 영역. 좋아요/팔로우/신고 시그널을 전달하면 추천 품질에 반영 (v3: 신고 시그널 추가)
