# AI UGC Platform 1 — 프로필 & 네비게이션

> 📋 UGC Platform PRD는 화면 단위로 3개로 분리되어 있으며, 순서대로 개발·테스트한다.
> 
> 1. 프로필 & 네비게이션 (본 문서) — 앱 골격 → 가장 먼저
> 2. 피드 인터랙션 & 페이백 — PRD 1 완료 후
> 3. 소셜 & 알림 — PRD 1 완료 후 (PRD 2와 병렬 가능)

## 교차 참조 규칙

- 팔로워/팔로잉 카운트 노출은 본 PRD에서 구현하되, 팔로우 기능 자체는 PRD 3에서 구현
- 좋아요 탭 껍데기는 본 PRD에서 구현하되, 좋아요 기능 자체는 PRD 2에서 구현
- 설정 화면의 차단 관리/알림 설정 메뉴 진입점은 본 PRD에서, 실제 기능은 PRD 3에서
- 타 유저 프로필 화면 구조는 본 PRD에서, 차단/신고 액션은 PRD 3에서

---

## Overview

앱의 네비게이션 골격(하단 탭 바 3탭)과 프로필 시스템(MY 3탭, 프로필 편집, 타 유저 프로필, 설정)을 구축한다.

### FE 재구성 방향

- 탭 1 — 홈: 그리드 피드. 헤더: 좌측 ZZEM 로고, 우측 크레딧 + 알림(종) 아이콘
- 탭 2 — 탐색(돋보기): 그리드 피드 숏컷. 홈과 동일한 콘텐츠 풀/추천 로직
- 탭 3 — 프로필: 기존 '내가 만든 밈' 버튼 대체
- MY 페이지: '필터/프롬프트' 2탭 → '게시물(공개)/비공개/좋아요' 3탭으로 완전 재구성
- 전면 리팩토링: PRD 기준으로 완전 재구성 (점진적 얹기 아님)

## User Stories & Acceptance Criteria

### US2: 프로필

유저로서, 내 프로필을 관리하고 내 콘텐츠를 모아볼 수 있으며, 다른 유저와 공유할 수 있다.

### AC 2.1: 프로필 진입점

- Given 유저가 로그인된 상태일 때
- When 하단 MY 탭을 탭하면
- Then 프로필 페이지로 이동한다
- Then 디폴트 랜딩 탭: 게시물(공개) 탭. 공개 콘텐츠가 없으면 비공개 탭. 둘 다 없으면 게시물 탭

### AC 2.2: 프로필 구조 및 탭

- Given 프로필 페이지에 진입하면
- Then 프로필 이미지, 닉네임, 팔로워/팔로잉/재생성된 카운트가 노출된다
- Then 3개 탭: 게시물(공개) / 비공개 / 좋아요
- Then "재생성된" = 내 공개 콘텐츠가 타 유저에 의해 재생성된 횟수 (전체 합산, 건별 카운트 — 동일 유저가 3번 재생성하면 3으로 카운트)
- Then 재생성 횟수 노출 정책: 프로필에서는 레벨 합산 노출 + 세로 스와이프 액션 바에서는 개별 콘텐츠별 노출

> ⚠️ 팔로워/팔로잉 카운트는 본 PRD에서 UI 노출만 구현. 팔로우 기능은 PRD 3에서 구현. PRD 1 단계에서는 카운트 0으로 표시.

### AC 2.3: 프로필 공유

- Given 프로필 페이지에서
- When "프로필 공유" 버튼을 탭하면
- Then OS 기본 공유 시트가 호출되며 프로필 URL(딥링크)이 포함된다
- Then 앱 설치 유저는 앱 내 프로필로, 미설치 유저는 브릿지 웹 경유
- Then 프로필 URL 공유 시 OG Image 미포함, 단순 딥링크로만 동작

### AC 2.4: 프로필 편집

- Given 프로필 페이지에서
- When "프로필 편집" 버튼을 탭하면
- Then 프로필 이미지와 닉네임을 수정할 수 있다
- Then 프로필 이미지 변경: 카메라 촬영 또는 앨범에서 선택
- Then 닉네임 변경은 제한 없이 언제든 자유롭게 변경 가능하다
- Then 닉네임 글자수: 최소 2자, 최대 20자
- Then 기본 프로필 이미지: 회색 디폴트 아바타 아이콘 (프로필 미설정 시 노출)

### AC 2.5: 프로필에서 세로 스와이프 진입

- Given 프로필의 게시물/비공개 탭에서
- When 콘텐츠를 탭하면
- Then 세로 스와이프 피드로 진입하며, 해당 탭의 콘텐츠만 노출된다

### AC 2.6: 닉네임 자동 생성 규칙

- Given 유저가 최초로 앱에 진입하여 프로필이 생성될 때
- Then 시스템이 자동으로 닉네임을 생성한다
- Then 자동 생성 포맷: 형용사 + 동물 + 4자리 숫자 (예: '빛나는고양이1234'). 불쾌한 조합 방지를 위해 사전 검수된 단어 목록 사용
- Then 유저는 나중에 프로필 편집에서 닉네임을 변경할 수 있다

### AC 2.7: 생성 후 프로필 랜딩

- Given 유저가 밈 생성을 요청한 후
- Then 프로필 화면으로 이동한다 (기존 '내가 만든 밈' 화면 대체)
- Then 필터 기반 생성 → 게시물 탭으로 랜딩
- Then 커스텀 프롬프트 기반 생성 → 비공개 탭으로 랜딩
- Then 생성 중/생성 실패 상태도 해당 탭에서 노출
- Then 이 규칙은 생성 직후 진입에만 적용되며, 일반 진입(MY 탭 등)은 AC 2.1 규칙을 따른다

### AC 2.8: 설정 화면

- Given 프로필 화면 우상단 톱니바퀴 아이콘을 탭하면
- Then 설정 화면으로 이동하며 다음 메뉴가 노출된다:
1. 프로필 편집
1. 알림 설정 → PRD 3에서 기능 구현
1. 차단 관리 (차단한 유저 리스트 + 차단 해제) → PRD 3에서 기능 구현
1. 서비스 이용약관
1. 개인정보 처리방침
1. 로그아웃
1. 탈퇴

> ⚠️ 알림 설정, 차단 관리는 본 PRD에서 메뉴 진입점만 구현. 탭 시 빈 화면 또는 "준비 중" 표시. PRD 3에서 실제 기능 연결.

---

### 타 유저 프로필 (US7에서 발취)

### AC 7.1: 타 유저 프로필 화면

- Given 세로 스와이프 피드의 프로필 영역(좌측 하단) 또는 팔로워/팔로잉 리스트에서 유저를 탭하면
- Then 해당 유저의 프로필 페이지로 이동
- Then 프로필 이미지, 닉네임, 팔로워/팔로잉/재생성된 카운트 노출
- Then 게시물 탭만 노출 (비공개/좋아요 탭 미노출)
- Then 더보기 메뉴: 프로필 URL 복사 (차단하기/신고하기는 PRD 3에서 추가)
- When 타유저 프로필의 게시물 탭에서 콘텐츠를 탭하면 → 세로 스와이프 피드로 진입, 해당 유저의 게시물만 노출

### AC 7.5: 페르소나 계정 처리 (프로필 관련)

- Then 페르소나 계정 = 콘텐츠 팩토리 운영용 데이터 전용 계정
- Then 계정 레벨 페르소나 여부 식별 플래그/필드 필요 (BE 구현)
- Then 페르소나 프로필 진입 시 생성 화면으로 이동하는 동작은 동일

## 비즈니스 룰

### 프로필 규칙

1. 닉네임 변경 제한 없음. 글자수 2~20자
1. 기본 프로필 이미지: 회색 디폴트 아바타 아이콘
1. 프로필 이미지 변경: 카메라 촬영 또는 앨범 선택
1. 숫자 포맷: 팔로워/팔로잉/재생성된 카운트 = 천/만 단위 축약 (8,600 → "8.6천")
1. 콘텐츠 정렬 순서: 내 프로필 전체 탭(3개) 최신순. 좋아요 탭은 좋아요 누른 시점 최신순

### 비회원 규칙

1. 비회원은 피드 탐색(홈)만 가능
1. 알림 탭, MY 탭, 피드 상세의 유저 프로필 탭 시 로그인 페이지로 랜딩
1. 좋아요, 팔로우, 생성/재생성, 더보기 등 인터랙션 버튼은 미노출 또는 탭 시 로그인 페이지로 랜딩

### 기존 밈 마이그레이션

1. 기존 생성 밈 → 비공개 탭으로 자동 분류
1. 마이그레이션 방식: 스키마 default만 (No Batch) — Content 스키마에 isPublished 필드를 default=false로 추가. 기존 콘텐츠는 필드 미존재 = 비공개로 간주
