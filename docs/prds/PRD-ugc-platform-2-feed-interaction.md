# AI UGC Platform 2 — 피드 인터랙션 & 페이백

> 📋 UGC Platform PRD는 화면 단위로 3개로 분리되어 있으며, 순서대로 개발·테스트한다.
> 
> 1. 프로필 & 네비게이션 — 앱 골격
> 2. 피드 인터랙션 & 페이백 (본 문서) — PRD 1 완료 후
> 3. 소셜 & 알림 — PRD 1 완료 후 (PRD 2와 병렬 가능)

## 교차 참조 규칙

- 좋아요 기능(토글, 카운트)은 본 PRD에서 구현. 좋아요 탭 UI 껍데기는 PRD 1에서 이미 구현됨
- 더보기 메뉴의 신고하기는 PRD 3에서 구현 (본 PRD에서는 내 콘텐츠 메뉴만)
- 페이백 적립 로직과 히스토리는 본 PRD. 페이백 알림(배치 푸시)은 PRD 3
---

## Overview

세로 스와이프 피드에서의 모든 인터랙션(공개/비공개, CTA, 좋아요, 재생성)과 크레딧 페이백 시스템을 구축한다.

### 구현 범위

- 콘텐츠 자동 공개 / 비공개 전환 (게시 토글)
- 커스텀 프롬프트 콘텐츠 공개 차단
- 세로 스와이프 액션 바 재구성 (좋아요, 재생성 수, 공유, 더보기)
- CTA 분기 (템플릿 사용하기 / 다시 생성하기)
- 재생성 플로우 + sourceContentId 추적
- 좋아요 토글 + 카운트 노출
- 크레딧 페이백 (1% 적립, 프로모션 크레딧, 소수점 올림)
- 페이백 안내 모달 (최초 1회) + 크레딧 히스토리 페이백 항목
---

## User Stories & Acceptance Criteria

### US1: 피드 공개

콘텐츠 생성자로서, 내 AI 생성 결과물을 피드에 공개하여 다른 유저들이 볼 수 있게 하고, 페이백 크레딧을 받고 싶다.

### AC 1.1: 공개 기본 정책

- Given 유저가 필터(프리셋)를 사용하여 콘텐츠를 생성 완료했을 때
- When 결과물이 생성되면
- Then 콘텐츠는 자동으로 공개(게시) 상태가 되며, 피드에 노출된다
- Then 콘텐츠에 워터마크/로고 없이 원본 그대로 노출된다
- Then 기능 업데이트 이전에 생성된 콘텐츠는 비공개 상태를 유지한다

### AC 1.2: 페이백 조건

- Given 콘텐츠가 공개 상태일 때
- When 해당 콘텐츠가 다른 유저에 의해 세로 스와이프 CTA('템플릿 사용하기')를 통해 재생성되면
- Then 원콘텐츠 소유자에게 재생성자가 소비한 크레딧의 1%가 페이백된다 (Config 환경변수로 비율 변경 가능)
- Then 재생성 시 sourceContentId를 생성 요청 파라미터에 포함하여 원본 콘텐츠와 페이백 대상을 추적
- Then Phase 1(MVP): 마진 체크 없이 1% 무조건 지급. 대시보드 모니터링으로 적자 감시
- Then 페이백 체인 규칙: 직전 1단계만 추적. A→B→C 재생성 체인에서 C가 재생성 시 B에게만 페이백
> ❗ 재생성 정의: 홈에서 직접 필터를 선택하여 생성하는 행위는 재생성으로 카운트되지 않는다. 오직 세로 스와이프 피드의 '템플릿 사용하기' CTA를 통해 진입한 생성만 재생성으로 카운트된다.

### AC 1.3: 비공개 전환

- Given 콘텐츠가 공개 상태일 때
- When 유저가 게시 토글을 OFF하면
- Then 비공개 전환 확인 바텀시트가 노출된다
- Then 확인 시 콘텐츠가 비공개로 전환되고 피드에서 제거된다

### AC 1.4: 커스텀 프롬프트 결과물

- Given 커스텀 프롬프트 기반으로 생성된 콘텐츠일 때
- Then 피드 공개(게시) 불가 — 프로필 비공개 탭에만 위치
- Then 세로 스와이프에서 게시 버튼 클릭 시 "커스텀 프롬프트 콘텐츠는 공개할 수 없습니다" 안내 노출
- Then Phase 2에서 커스텀 프롬프트 기반 UGC 피드 공개 지원 예정
- Then 비공개 탭에서 프롬프트 편집하기 CTA 버튼의 크레딧은 최초 생성 시 선택한 화질 기준으로 산정

### AC 1.5: 풀스크린 비율 정책

- Given 피드 콘텐츠가 풀스크린으로 표시될 때
- Then 지원 비율: 4:5, 9:16 등 세로형 비율
- Then 풀스크린 표시 방식: cover (크롭) — 콘텐츠가 화면을 꽉 채우며, 비율이 맞지 않으면 크롭
- Then 16:9 디바이스에서 세로형(9:16) 콘텐츠 렌더링 시 좌우를 크롭
- Then 세로형 피드에서 좌우 스와이프 시 이미지 간 이격(갭) 허용

### AC 1.6: 피드 CTA 버튼 분기

- Given 세로 스와이프 피드에서 콘텐츠 하단 CTA 버튼이 표시될 때
- When 타인의 콘텐츠를 보고 있으면 → CTA = "템플릿 사용하기" — 해당 필터/프리셋으로 새 콘텐츠 생성 화면 진입
- When 내 콘텐츠를 보고 있으면 → CTA = "다시 생성하기" — 동일 필터/프리셋으로 재생성
- Then 시스템은 콘텐츠 소유자 ID와 현재 유저 ID를 비교하여 CTA 유형을 분기한다

### AC 1.7: 재생성 플로우

- Given 세로 스와이프 피드에서 [다시 생성하기]를 탭하면
- Then 해당 필터의 생성 피드로 이동하여 유저가 직접 이미지를 등록하고 생성한다 (프리뷰 화면 없음)
- **단, 믹스드 템플릿(MIXED_IMAGE_TO_VIDEO, MIXED_IMAGE_TO_IMAGE 타입)은 예외 — 반드시 사진 업로드 화면(프리뷰)을 거친다**
- When 템플릿 사용하기 시 해당 필터가 삭제된 경우 → "해당 필터가 삭제되었습니다" 모달 노출

### AC 1.8: 세로 스와이프 액션 바 및 더보기 메뉴

- Given 세로 스와이프 피드에서 콘텐츠를 보고 있을 때
- Then 우측 액션 바에 다음 버튼이 위에서 아래로 배치: 1) 좋아요(하트) — 좋아요 수 표시(실제 숫자, 축약 없음) 2) 재생성 — 해당 콘텐츠의 재생성 횟수 표시 3) 공유하기 — OS 기본 공유 시트 호출 (공유 횟수는 미노출) 4) 더보기(···) — 바텀시트 메뉴 호출
- Then 더보기 메뉴는 콘텐츠 소유자에 따라 분기: 내 콘텐츠 = 다운로드 / 의견 보내기 / 삭제 | 타 유저 콘텐츠 = 다운로드 / 의견 보내기 / 신고하기(→PRD 3)
- Then 삭제 선택 시 삭제 확인 팝업이 노출된다
- Then 의견 보내기는 기존 구현된 기능과 동일 (100자 이내 자유 입력)
- Then 콘텐츠 삭제는 세로 스와이프 더보기에서만 가능 (프로필 그리드에서는 삭제 불가)
> ⚠️ 댓글 기능은 Spec-out. Figma에 디자인 존재하나 Phase 1에서 미구현. 액션 바에서 댓글 버튼 미노출.

### AC 1.9: 게시 토글

- Given 세로 스와이프 피드에서 내 콘텐츠를 보고 있을 때
- Then 하단 CTA 바에 "게시" 토글 버튼이 노출된다 (타 유저 콘텐츠에서는 미노출)
- When 게시 토글을 ON→OFF (비공개 전환)하면 → AC 1.3의 비공개 전환 확인 바텀시트 노출
- When 게시 토글을 OFF→ON (공개 전환)하면 → 즉시 공개 상태로 전환되어 피드에 노출

### US3: 좋아요

콘텐츠 소비자로서, 마음에 드는 콘텐츠에 좋아요를 누르고 나중에 모아볼 수 있다.

### AC 3.1: 셀프 좋아요

- Given 내 콘텐츠의 세로 스와이프 피드에서
- When 좋아요 버튼을 탭하면
- Then 셀프 좋아요가 허용되며 좋아요 카운트에 반영된다

### AC 3.2: 좋아요 탭

- Given 마이페이지 > 좋아요 탭에서
- Then 좋아요한 콘텐츠를 좋아요 누른 시점 최신순으로 모아볼 수 있다
- Then 피드, 프로필, 검색 등 어떤 경로에서 누른 좋아요든 단일 탭에 통합 표시

### AC 3.3: 좋아요 표시

- Then 좋아요 수 표시: 그리드 카드 + 세로 스와이프 상세 모두에서 실제 숫자 그대로 노출 (축약 없음, 0일 때도 노출)
- Then 숫자 포맷 규칙: 좋아요 수 = 실제 숫자(축약 없음). 팔로워/팔로잉/재생성된 카운트 = 천/만 단위 축약 (8,600 → "8.6천")
- Then 좋아요 데이터는 추천 시스템에 시그널로 전달된다
---

### US4: 크레딧 페이백

콘텐츠 생성자로서, 내 공개 콘텐츠가 재생성될 때 크레딧을 페이백 받아 더 많은 콘텐츠를 만들고 싶다.

### AC 4.1: 페이백 조건

- Given 내 공개 콘텐츠가 타 유저에 의해 세로 스와이프 CTA('템플릿 사용하기')를 통해 재생성될 때
- Then 재생성자가 소비한 크레딧의 1%가 내 계정에 적립 (Config 환경변수로 변경 가능)
- Then 재생성 시 sourceContentId 파라미터로 원본 콘텐츠를 추적
- Then Phase 1(MVP): 마진 체크 없이 1% 무조건 지급
- Then 페르소나 계정 콘텐츠가 재생성될 경우 페이백 적립 대상 없음 (수신 계정 없음)

### AC 4.2: 크레딧 히스토리

- Given 페이백이 발생하면
- Then 크레딧 히스토리에 "크레딧 페이백" 타이틀 + 콘텐츠 썸네일로 노출

### AC 4.3: 크레딧 페이백 세부 정책

- Then 크레딧 타입 = 프로모션 크레딧 (유료 크레딧과 구분)
- Then 소수점 처리 = 올림 후 점수 단위 (예: 1.2크레딧 → 2크레딧 적립)
- Then Phase 1(MVP): 마진 계산 로직 없이 1% 무조건 지급. Phase 2에서 마진 공식 적용
- Then Phase 2 적용 시: 마진 압박 시 최저 0.1% 하한까지 축소 가능 (Config로 조정)

### AC 4.4: 최초 진입 UX (페이백 안내)

- Given 유저가 콘텐츠를 최초로 생성하고 공개했을 때
- Then 1% 페이백 안내 모달이 노출: "콘텐츠가 재생성될 때마다 생성 비용의 1%가 페이백됩니다"
- Then 모달 확인 후 피드 전환 CTA 노출 (피드 탐색 유도)
- Then 1회성 노출 (최초 공개 시에만)

## 비즈니스 룰

### 콘텐츠 공개 규칙

1. 커스텀 프롬프트 결과물 = 피드 공개 불가 (Phase 2 예정)
2. 기능 업데이트 전 콘텐츠 = 비공개 상태 유지
3. 업데이트 후 콘텐츠 = 공개가 디폴트
4. 콘텐츠 공개 시 워터마크/로고 없이 그대로 노출
5. 세로 스와이프 상단에 '✨ AI가 만드는 컨텐츠예요' 라벨 항상 노출. 투명성 정책으로 제거 불가

### 페이백 규칙

1. 페이백 비율 = 재생성자가 소비한 크레딧의 1% (Config/환경변수로 변경 가능)
2. Phase 1(MVP): 마진 체크 없이 1% 무조건 지급. 대시보드 모니터링으로 적자 감시
3. 재생성 정의: 세로 스와이프 CTA('템플릿 사용하기') 경유 생성만 재생성으로 카운트
4. 추적 방식: 재생성 시 sourceContentId를 생성 요청 파라미터에 포함
5. 체인 규칙: 직전 1단계만 추적. A→B→C에서 C 재생성 시 B에게만 페이백
6. 공개→비공개 전환 시 페이백: 회수 안 함. 한 번 지급된 페이백 크레딧은 유지
7. 비공개→공개 재전환 시: 재공개 후 재생성 시 페이백 다시 발생. 토글 어뷰징 모니터링 필요
8. 마진 부족 시: 시스템 자동 차단 없음. 대시보드 모니터링 후 Config 값 수동 조정
9. 공개 콘텐츠 삭제: 삭제 가능. 이미 지급된 페이백 크레딧은 유지
10. 어드민 크레딧 회수: 어드민 패널에서 수동 회수 가능. 히스토리에 '크레딧 회수' 노출
11. 페르소나 콘텐츠 재생성 시 페이백 적립 대상 없음 (수신 계정 없음)

### 좋아요 규칙

1. 셀프 좋아요 허용
2. 좋아요 수 표시: 실제 숫자 그대로 노출 (축약 없음, 0일 때도 노출)
3. 페르소나 콘텐츠 좋아요: 가능

## 경계 (3-Tier Boundary)

### ALWAYS DO

1. 콘텐츠 공개/비공개 전환 시 확인 바텀시트 노출
2. Phase 1에서 페이백 1% 무조건 지급
3. 커스텀 프롬프트 콘텐츠는 공개 불가 처리
4. 기존 크레딧 시스템 API를 통한 적립/차감
5. 공개→비공개 전환 시 페이백 크레딧 회수 안 함
6. AI 콘텐츠 라벨 항상 노출

### NEVER DO
