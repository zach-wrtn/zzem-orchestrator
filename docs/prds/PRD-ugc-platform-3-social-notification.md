# AI UGC Platform 3 — 소셜 & 알림

> 📋 UGC Platform PRD는 화면 단위로 3개로 분리되어 있으며, 순서대로 개발·테스트한다.
>
> 1. 프로필 & 네비게이션 — 앱 골격
> 2. 피드 인터랙션 & 페이백 — PRD 1 완료 후
> 3. 소셜 & 알림 (본 문서) — PRD 1 완료 후 (PRD 2와 병렬 가능)

## 교차 참조 규칙

- 팔로워/팔로잉 카운트 UI는 PRD 1에서 이미 구현됨. 본 PRD에서 팔로우 기능 연결
- 설정 화면의 알림 설정/차단 관리 진입점은 PRD 1에서 구현됨. 본 PRD에서 실제 기능 연결
- 좋아요 알림은 PRD 2의 좋아요 기능이 전제. 페이백 알림은 PRD 2의 페이백 적립이 전제
- 타 유저 프로필 더보기의 차단/신고는 본 PRD에서 추가

---

## Overview

유저 간 소셜 관계(팔로우, 차단, 신고)와 알림 시스템(알림센터, 푸시, 설정)을 구축한다.

### 구현 범위

- 팔로우/언팔로우 + 팔로워·팔로잉 리스트
- 차단 (양방향) + 차단 해제 + 차단 관리
- 신고 (자유 텍스트) + 추천 시스템 시그널 전달
- 알림센터 (시간순 리스트, 카테고리 색상 dot, Red dot)
- 푸시 알림 3종 (좋아요 실시간, 팔로우 실시간, 페이백 배치)
- 알림 설정 토글 4종
- 페르소나 계정 알림/푸시 제외

---

## User Stories & Acceptance Criteria

### US6: 팔로우

유저로서, 마음에 드는 크리에이터를 팔로우하여 그들의 새 콘텐츠를 피드에서 더 쉽게 발견하고 싶다.

### AC 6.1: 팔로우 상태

- Then 팔로우 상태는 팔로우/팔로잉/맞팔로우 3가지로 표시된다

### AC 6.2: 팔로우 리스트

- Given 내 프로필에서
- When 팔로워/팔로잉 카운트를 탭하면
- Then 각각의 리스트가 가나다순으로 노출되며, 각 유저에 팔로우 상태 버튼 노출
- Then 타 유저 프로필에서는 팔로워/팔로잉 숫자만 노출 (리스트 조회 불가, 본인만 가능)

### AC 6.3: 팔로우와 피드 추천

- Given 유저가 특정 크리에이터를 팔로우하면
- Then 해당 크리에이터의 콘텐츠에 피드 추천 가중치(부스트)가 부여된다
- Then 상세 로직은 PA(추천) 팀에서 정의 (별도 전달 필요)

---

### US7: 차단 & 신고

### AC 7.2: 차단

- Given 유저 A가 유저 B를 차단하면
- Then 차단 확인 바텀시트 노출 후 차단 처리
- Then 양방향 차단: A와 B 서로의 프로필/콘텐츠 접근 불가. A의 피드에서 B 콘텐츠 미노출, B의 피드에서 A 콘텐츠 미노출
- Then 팔로우 양방향 해제: A→B, B→A 팔로우 모두 해제
- Then 차단 사실은 상대방에게 미통지
- Then 차단 해제는 언제든 가능
- Then 차단된 프로필 진입 시: "이 계정을 차단했어요" + 차단 해제 버튼
- Then 차단 시 좋아요 데이터: DB에 유지, 노출만 차단. 차단 해제 시 다시 보임

### AC 7.3: 신고

- Then 신고 사유: 자유 텍스트 입력만 (100자 이내, 필수). 사유 선택 드롭다운 없음
- Then 신고 사실은 상대방에게 미통지

### AC 7.4: 신고 → 추천 알고리즘 연동

- Given 콘텐츠가 신고를 받았을 때
- Then 신고한 유저에게는 해당 콘텐츠 미노출 (개인화 필터링)
- Then 개인화 필터링 반영 지연: ~1시간 (실시간 아님)

> ⚠️ 신고 비율 임계값, 추천 패널티 로직, 노출 가중치 하향은 전적으로 PA(추천) 팀 소관. 개발단에서 신고 데이터를 추천 시스템에 시그널로 전달하는 것까지만 구현.

### AC 7.5: 페르소나 계정 처리 (소셜 관련)

- Then 페르소나 계정은 알림 수신 불가, 페이백 적립 대상 없음
- Then 유저가 페르소나를 팔로우 가능 (UX 상 구분 없음), 단 페르소나 측에 팔로우 알림 발송 안 함
- Then 페르소나 콘텐츠 좋아요: 가능

---

### US5: 알림 & 푸시

유저로서, 내 콘텐츠에 대한 좋아요, 페이백, 팔로우 등의 알림을 받아 소식을 확인하고 싶다.

### AC 5.1: 알림 카테고리

- Then 좋아요 알림, 크레딧 알림, 소식 알림, 팔로우 알림 카테고리가 제공된다 (디폴트 전부 ON)
- Then 좋아요 알림: 빈도 제한 없음 (개별 알림)
- Then 팔로우 알림: 건별 발송, 맞팔로우 시 별도 알림 없음. 빈도 제한 없음
- Then 크레딧 페이백 알림: 배치 발송 — 매일 1회 오전 10시 KST. 전일 발생한 페이백을 합산하여 한 번에 알림
- Then 소식 알림: 기 구현 완료 — 본 PRD 범위 밖
- Then 페르소나 계정은 푸시/알림센터 발송 대상에서 제외

### AC 5.2: 팔로우 알림

- Given 다른 유저가 나를 팔로우했을 때
- Then 즉시 푸시 알림 + 인앱 알림센터에 노출
- Then '활동 알림' 카테고리에 포함
- Then 알림 내용: 'OOO님이 회원님을 팔로우했습니다'

### AC 5.3: 알림센터 공통

- Then 알림센터 진입점: 홈 헤더 우상단 종(벨) 아이콘
- Then 알림센터 보관: 1개월. 하단에 "1개월 전 알림까지 확인할 수 있어요" 안내 노출
- Then Red dot 로직: 미확인 알림 존재 시 종 아이콘에 빨간점 노출. 알림센터 진입(탭) 시 일괄 삭제
- Then 알림센터 UI: 탭 필터 없이 시간순 단일 리스트. 각 알림에 카테고리별 색상 dot 구분 (소식=보라, 좋아요=분홍, 크레딧=초록, 팔로우=분홍)
- Then 빈 상태: 알림이 없을 때 "아직 도착한 알림이 없어요" 메시지 노출
- Then 푸시 권한 배너: 디바이스 알림이 꺼져있을 때 알림센터 상단에 "휴대폰의 앱 알림이 꺼져있어요" + "알림 켜기" 버튼 노출
- Then 알림 탭 동작: 좋아요→해당 콘텐츠 세로 스와이프, 팔로우→해당 유저 프로필, 크레딧→크레딧 히스토리, 소식→기존 동작 유지

### AC 5.4: 푸시 알림 메시지 템플릿

기존 new-noti 서버 + UserDevice.fcmToken 인프라를 통해 발송. PushNotificationPayload 인터페이스 사용.

- 좋아요: 실시간 개별 | 타이틀: "{nickname}님이 회원님의 콘텐츠를 좋아합니다" | 딥링크: 해당 콘텐츠 세로 스와이프
- 팔로우: 실시간 건별 | 타이틀: "{nickname}님이 회원님을 팔로우했습니다" | 딥링크: 해당 유저 프로필
- 크레딧 페이백: 매일 1회 10시 KST 배치 | 타이틀: "어제 {N}크레딧이 페이백되었어요!" | 바디: "회원님의 콘텐츠가 {M}회 재생성되었습니다" | 딥링크: 크레딧 히스토리
- Fallback 딥링크: 파싱 실패 시 홈 화면으로 랜딩

### AC 5.5: 알림 설정

- Given 알림 설정 화면에 진입하면
- Then 4개 토글: 1) 푸시 알림 — 전체 ON/OFF 2) 좋아요 알림 ON/OFF 3) 소식 알림 ON/OFF 4) 팔로우 알림 ON/OFF (디폴트 전부 ON)
- Then 크레딧 페이백 알림은 별도 토글 없이 항상 ON (페이백 정보는 반드시 전달)
- Then 푸시 알림 전체 OFF 시 하위 개별 토글도 비활성화

## 비즈니스 룰

### 팔로우 규칙

1. 차단 시 팔로우 양방향 해제
2. 팔로우한 사람의 콘텐츠 = 피드 추천에 부스트 반영
3. 페르소나 계정 팔로우: 가능. 팔로우 알림은 발송 안 함

### 차단 규칙

1. 양방향 차단: 차단 시 A와 B 서로의 콘텐츠가 피드/프로필에서 미노출. 차단 해제 시 다시 보임
2. 차단 시 좋아요 데이터: DB에 유지, 노출만 차단
3. 차단/차단 해제 사실은 상대방에게 미통지

### 알림 규칙

1. 알림센터 보관: 1개월
2. Red dot: 미확인 알림 시 표시. 알림센터 진입 시 일괄 삭제
3. 페르소나 계정은 모든 알림 수신 대상에서 제외
4. 크레딧 페이백 알림: 배치 발송 — 매일 1회 오전 10시 KST. 별도 토글 없이 항상 ON
5. 푸시 발송 인프라: 기존 new-noti 서버 + UserDevice.fcmToken 연동

## 경계 (3-Tier Boundary)

### ALWAYS DO

1. 차단/신고 사실은 상대방에게 미통지
2. 페르소나 계정은 알림/페이백 대상 제외
3. 차단 시 팔로우 양방향 해제

### NEVER DO

1. 팔로워 피드/타임라인 구현 (추천 피드에 팔로우 부스트만 반영)
2. 추천 알고리즘 코어 로직 직접 변경 (시그널 전달만)
3. 신고 비율 임계값/패널티 가중치 로직 구현 (PA 추천팀 소관)
4. 뮤트 기능 (추후 검토)
