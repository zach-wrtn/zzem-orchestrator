# 1. Overview

## 한 줄 요약

- 쨈(Zzem) 앱의 홈 화면으로, 사용자가 AI 기반 콘텐츠 생성 템플릿을 탐색·발견하고 빠르게 제작에 진입할 수 있는 허브 역할을 한다.

## Why (비즈니스 목표)

<!-- 확인 필요 -->

1. 콘텐츠 생성 전환율 향상 — 홈에서 템플릿 발견 → 만들기 진입까지의 퍼널을 최소화
2. 사용자 리텐션 강화 — 실시간 랭킹, 카테고리 필터, 신규 템플릿 노출로 재방문 동기 부여
3. 신규 기능 채택률 제고 — 퀵 액션 메뉴와 Beta 배지를 통해 신규 기능(주인공 바꾸기 등) 노출

## 대상 사용자

<!-- 확인 필요 -->

- 숏폼/밈 콘텐츠를 즐기는 일반 사용자 (10~30대)
- AI 기반 이미지/비디오 생성에 관심 있는 크리에이터
- 트렌드에 민감하고 챌린지 참여에 적극적인 소셜 미디어 사용자

## KPI 기여 논리

<!-- 확인 필요 -->

- 홈 → 템플릿 상세 클릭률 → 콘텐츠 생성 완료율 → DAU/MAU 증가
- 퀵 액션 메뉴 사용률 → 기능별 생성 전환율 → 기능 채택률
- 실시간 랭킹 "만들기" 클릭률 → 인기 템플릿 생성 전환율 → 콘텐츠 생산량 증가
- 필터 칩 사용률 → 탐색 깊이(세션당 뷰 수) → 체류 시간 증가

## 선행 조건

- AI 콘텐츠 생성 엔진 (비디오 생성, 이미지 생성, 댄스 챌린지, 주인공 바꾸기)
- 템플릿 관리 시스템 (CRUD, 카테고리 분류, 랭킹 집계)
- 사용자 인증/프로필 시스템 (좋아요, 크리에이터 프로필)
- 코인/재화 시스템 (헤더의 코인 아이콘)
- 푸시 알림 시스템 (헤더 알림 벨 + My 탭 알림 뱃지)

# 2. User Stories & Acceptance Criteria

## US1: 홈 화면 진입 및 콘텐츠 탐색

사용자로서, 앱을 열면 홈 화면에서 다양한 AI 생성 콘텐츠 템플릿을 한눈에 확인하여 만들고 싶은 콘텐츠를 빠르게 찾고 싶다.

### AC 1.1: 홈 화면 기본 로드

- Given 인증된 사용자가 앱을 실행하면
- When 홈 화면이 로드될 때
- Then 헤더(로고, 코인, 알림), 퀵 액션 메뉴(4개), 신규 템플릿, 실시간 랭킹, 필터 칩, 콘텐츠 그리드가 순서대로 표시된다

### AC 1.2: 알림 뱃지 표시

- Given 읽지 않은 알림이 있을 때
- When 홈 화면을 볼 때
- Then 헤더의 알림 벨 아이콘과 하단 My 탭에 빨간 알림 점이 표시된다

---

## US2: 퀵 액션을 통한 콘텐츠 생성 진입

사용자로서, 홈 상단의 빠른 메뉴를 통해 원하는 유형의 콘텐츠 생성 화면으로 바로 이동하고 싶다.

### AC 2.1: 퀵 액션 메뉴 표시

- Given 홈 화면이 로드되면
- When 퀵 액션 영역을 볼 때
- Then 비디오 생성, 이미지 생성, 댄스 챌린지, 주인공 바꾸기 4개의 버튼이 아이콘+라벨로 표시된다

### AC 2.2: 퀵 액션 탭

- Given 퀵 액션 메뉴가 표시된 상태에서
- When 특정 액션 버튼(예: 비디오 생성)을 탭하면
- Then 해당 기능의 생성 화면으로 이동한다

### AC 2.3: Beta 기능 표시

- Given 주인공 바꾸기 기능이 Beta 상태일 때
- When 퀵 액션 메뉴를 볼 때
- Then 해당 버튼 위에 보라색 "Beta" 툴팁 배지가 표시된다

---

## US3: 신규 템플릿 탐색

사용자로서, 최근 추가된 새로운 템플릿을 빠르게 확인하여 트렌디한 콘텐츠를 만들고 싶다.

### AC 3.1: 신규 템플릿 섹션 표시

- Given 홈 화면이 로드되면
- When "쨈 신규 템플릿" 섹션을 볼 때
- Then 가로 스크롤 가능한 템플릿 카드 리스트(80×100px 썸네일 + 이름)가 표시된다

### AC 3.2: 가로 스크롤

- Given 신규 템플릿이 5개 이상일 때
- When 좌우로 스와이프하면
- Then 숨겨진 템플릿 카드가 나타난다

### AC 3.3: 전체보기 이동

- Given "쨈 신규 템플릿" 섹션 헤더의 우측 화살표를 탭하면
- When 탭 이벤트가 발생하면
- Then 신규 템플릿 전체 목록 화면으로 이동한다

---

## US4: 실시간 랭킹 확인 및 인기 템플릿 제작

사용자로서, 지금 인기 있는 템플릿을 확인하고 바로 만들기를 시작하고 싶다.

### AC 4.1: 랭킹 목록 표시

- Given 홈 화면이 로드되면
- When "실시간 랭킹" 섹션을 볼 때
- Then 1~3위 템플릿이 순위 번호(보라색), 썸네일, 이름, 콘텐츠 유형 태그, "만들기" 버튼과 함께 표시된다

### AC 4.2: 만들기 버튼 탭

- Given 랭킹 항목이 표시된 상태에서
- When "만들기" 버튼을 탭하면
- Then 해당 템플릿의 콘텐츠 생성 화면으로 이동한다

### AC 4.3: 더보기 확장

- Given 랭킹 3위까지 표시된 상태에서
- When "더보기" 링크를 탭하면
- Then 추가 랭킹 항목이 펼쳐져 표시된다

---

## US5: 카테고리 필터를 통한 콘텐츠 탐색

사용자로서, 원하는 카테고리의 콘텐츠만 필터링하여 관심 있는 템플릿을 쉽게 찾고 싶다.

### AC 5.1: 필터 칩 표시

- Given 홈 화면의 콘텐츠 그리드 상단에
- When 필터 영역을 볼 때
- Then "추천"(기본 선택), "무료", "비디오", "이미지", "조합", "말하는 아기" 등 1차 필터와 "연인", "반려동물", "시간여행", "재미", "댄스챌린지", "아트", "단체" 등 2차 필터가 가로 스크롤로 표시된다

### AC 5.2: 필터 선택/해제

- Given 필터 칩이 표시된 상태에서
- When 특정 칩(예: "무료")을 탭하면
- Then 해당 칩이 선택 상태(채워진 배경)로 변경되고 콘텐츠 그리드가 필터링된 결과로 갱신된다

### AC 5.3: 필터 가로 스크롤

- Given 필터 칩이 화면 너비를 초과할 때
- When 좌우로 스와이프하면
- Then 숨겨진 필터 칩이 나타난다

---

## US6: 콘텐츠 그리드 탐색 및 상호작용

사용자로서, 다양한 AI 생성 콘텐츠를 둘러보고 마음에 드는 것에 좋아요를 누르거나 제작을 시작하고 싶다.

### AC 6.1: 매거진형 그리드 표시

- Given 홈 화면 하단 콘텐츠 영역에
- When 콘텐츠 그리드를 볼 때
- Then 2열 매거진(Pinterest) 레이아웃으로 썸네일 카드가 표시되며, 각 카드에는 이미지, 템플릿 이름, 크리에이터 프로필(아바타+이름), 좋아요 버튼+카운트가 포함된다

### AC 6.2: 좋아요 토글

- Given 콘텐츠 카드가 표시된 상태에서
- When 하트 아이콘을 탭하면
- Then 좋아요 상태가 토글되고 카운트가 즉시 반영된다

### AC 6.3: 카드 탭

- Given 콘텐츠 카드가 표시된 상태에서
- When 카드 영역을 탭하면
- Then 해당 템플릿의 상세/생성 화면으로 이동한다

### AC 6.4: 무한 스크롤

<!-- 확인 필요 -->

- Given 콘텐츠 그리드가 표시된 상태에서
- When 하단까지 스크롤하면
- Then 추가 콘텐츠가 자동으로 로드된다

---

## US7: 하단 네비게이션을 통한 화면 전환

사용자로서, 하단 탭을 통해 홈, 검색, 마이페이지 간에 쉽게 전환하고 싶다.

### AC 7.1: 하단 네비게이션 표시

- Given 홈 화면에서
- When 하단 영역을 볼 때
- Then 홈(활성), 검색, My 3개 탭이 아이콘과 함께 표시되며, 현재 탭(홈)은 채워진 아이콘으로 구분된다

### AC 7.2: 탭 전환

- Given 하단 네비게이션이 표시된 상태에서
- When 비활성 탭(예: 검색)을 탭하면
- Then 해당 화면으로 전환되며 탭 아이콘 상태가 변경된다

# 3. Boundary & Constraints

## 3-Tier Boundary

| Tier | 항목 | 설명 |
| --- | --- | --- |
| Must | 퀵 액션 메뉴 (4종) | 비디오 생성, 이미지 생성, 댄스 챌린지, 주인공 바꾸기 |
| Must | 신규 템플릿 가로 스크롤 | 최소 5개 카드, 우측 화살표로 전체보기 이동 |
| Must | 실시간 랭킹 (Top 3) | 순위, 썸네일, 이름, 태그, 만들기 버튼, 더보기 |
| Must | 카테고리 필터 칩 | 1차(추천/무료/비디오/이미지/조합/말하는아기) + 2차(연인/반려동물/시간여행 등) |
| Must | 2열 매거진 콘텐츠 그리드 | 썸네일, 템플릿명, 크리에이터, 좋아요 |
| Must | 하단 네비게이션 (3탭) | 홈, 검색, My + 알림 뱃지 |
| Must | 헤더 | 로고, 코인 버튼, 알림 벨 (뱃지 포함) |
| Should | 다크모드 지원 | WDS 토큰에 Light/Dark 테마 정의 존재, Figma 디자인은 Light만 포함 |
| Should | 스켈레톤 로딩 | WDS에 skeleton 컴포넌트 토큰 정의됨, 디자인에는 미포함 |
| Should | 풀다운 새로고침 | 홈 피드 갱신 인터랙션 (디자인 미포함) <!-- 확인 필요 --> |
| Won't (this version) | 데스크톱/태블릿 반응형 | 디자인이 375px 모바일만 포함 <!-- 확인 필요 --> |
| Won't (this version) | 비로그인 사용자 홈 | 디자인에 미인증 상태 화면 없음 <!-- 확인 필요 --> |

| 1 | 신규 템플릿이 0건인 경우 | 섹션 숨김 또는 "곧 새로운 템플릿이 추가됩니다" 빈 상태 표시 <!-- 확인 필요 --> |

| 5 | 썸네일 이미지 로드 실패 | 플레이스홀더 이미지 표시 <!-- 확인 필요 --> |
| 6 | 좋아요 API 실패 | 낙관적 UI 롤백 처리 <!-- 확인 필요 --> |
| 7 | 매우 긴 템플릿/크리에이터 이름 | 텍스트 말줄임(...) 처리 |
| 8 | 코인 잔액 0 또는 유료 템플릿 접근 | 코인 충전 유도 화면 이동 <!-- 확인 필요 --> |
| 9 | Beta 기능(주인공 바꾸기) 비활성화/종료 시 | 퀵 액션에서 숨김 또는 "준비 중" 상태 표시 <!-- 확인 필요 --> |

# 5. Out of Scope

- 검색 화면 UI 및 검색 기능 (별도 PRD)
- 마이페이지(My) 화면 (별도 PRD)
- 각 콘텐츠 생성 화면 (비디오/이미지/댄스/주인공바꾸기) 상세 플로우 (별도 PRD)
- 템플릿 상세 화면 (별도 PRD)
- 알림 목록/상세 화면 (별도 PRD)
- 코인 충전/거래 내역 화면 (별도 PRD)
- 관리자 템플릿 관리/랭킹 운영 도구
- 데스크톱/태블릿 반응형 레이아웃
- 비로그인 사용자 경험
