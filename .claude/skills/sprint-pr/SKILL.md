# Sprint PR

## Goal

스프린트 실행 완료 후, 각 프로젝트 레포에서 sprint 브랜치 → base branch로 PR을 생성한다.
브랜치 전략은 `docs/designs/branch-strategy.md`를 따른다.

## Invocation

```
/sprint-pr <sprint-id>
/sprint-pr <sprint-id> --dry-run   # PR 생성 없이 미리보기만
```

## Workflow

### 1. 사전 확인

- sprint-config.yaml 로드
- 각 레포에 sprint 브랜치(`zzem/{sprint-id}`)가 존재하는지 확인
- sprint 브랜치에 base branch 대비 변경사항이 있는지 확인

### 2. 스프린트 상태 수집

각 태스크의 result 파일을 읽어 상태를 수집한다:
- COMPLETED / PARTIAL / FAILED 분류
- 변경된 파일 목록 수집

### 3. PR 생성 (레포별)

각 프로젝트 레포에 대해:

#### 3a. 변경사항 확인

```bash
cd {project-dir}
git checkout zzem/{sprint-id}
git log origin/{base-branch}..zzem/{sprint-id} --oneline
```

변경사항이 없으면 스킵.

#### 3b. Push

```bash
git push -u origin zzem/{sprint-id}
```

#### 3c. PR 생성

```bash
gh pr create \
  --base {base-branch} \
  --head zzem/{sprint-id} \
  --title "feat: Sprint {sprint-id} — {repo-name}" \
  --body "{PR body}"
```

**PR Body 형식:**

```markdown
## Sprint: {sprint-id}

### Summary
{PRD 스코프 요약 — PRD.md에서 추출}

### Branch Strategy
- **Base**: `{base-branch}`
- **Sprint Branch**: `zzem/{sprint-id}`
- **Merge Strategy**: task branches → sprint branch → PR

### Tasks
| Task | Status | Key Changes |
|------|--------|-------------|
| {task-id} | {status} | {변경 요약} |
| ... | ... | ... |

### Merge History
{git log --oneline 출력}

### QA Summary
- Total: {N} tasks
- Completed: {N}
- Failed: {N}

### Generated by
zzem-orchestrator Agentic Sprint System
```

### 4. 결과 출력

```
Sprint PR: {sprint-id}

  wrtn-backend:
    PR: {url}
    zzem/{sprint-id} → {base-branch}
    Changes: {N} commits, {N} files

  app-core-packages:
    PR: {url}
    zzem/{sprint-id} → {base-branch}
    Changes: {N} commits, {N} files
```

## Dry Run Mode

`--dry-run` 시:
- PR body를 출력만 하고 실제 생성하지 않음
- push하지 않음
- 변경사항 요약만 출력

## Constraints

- PR 생성 전 반드시 사용자에게 확인을 받는다 (push + PR은 외부 영향 있는 작업)
- 이미 동일 head/base로 열린 PR이 있으면 새로 만들지 않고 기존 PR URL을 안내
- sprint 브랜치에 FAILED 태스크가 있으면 PR body에 명시적으로 표기
